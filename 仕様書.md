# 工事進捗工程表管理システム 仕様書

## 1. 概要
本システムは、建設工事における各案件の進捗工程をデジタル化し、動的なガントチャートで可視化するWebアプリケーションである。

## 2. 機能構成

### 2.1 案件（物件）管理
- 物件情報の切り替え・新規作成・編集・削除機能。
- 保存データ項目:
  - タイトル
  - サブタイトル
  - 工事番号
  - 版（Revision）
  - 案件（Case）リスト（各案件の工事項目、カラー設定など）

### 2.2 進捗可視化（ガントチャート）
- 指定された開始日から計算された工程をガントチャート形式で表示。
- 土日祝日（または非稼働日）を考慮した日付管理。
- 凡例（Legend）によるステータス識別。

### 2.3 データ永続化
- **データベース**: Turso (SQLite)
- **認証**: API Token認証
- `useProperties` カスタムフックにより、データのロード・保存・削除を抽象化。

## 3. コンポーネント構成
- `ControlPanel`: 日付選択、設定モーダルの起動。
- `SettingsModal`: 物件設定の編集・保存。
- `CaseCard`: 物件単位の概要表示。
- `GanttChart`: 工程表本体。
- `SummaryCard`: 統計情報の表示。
- `Footer`: 備考および凡例の表示。

## 4. データ構造 (Types)
- `ScheduleConfig`: 全体の設定オブジェクト。
- `CaseConfig`: 個別の工事項目設定。
- `TaskConfig`: 工事項目内の詳細タスク。
- `Property`: DBに保存される単位。

## 5. デザインシステム
- **カラーパレット**: 
  - メインカラー: `#1b3558` (深い紺色)
  - サブカラー: `#9a9790` (グレーベージュ)
  - 背景色: `#f2efea` (温かみのある白)
- **タイポグラフィ**: `DM Mono`, `sans-serif`
- **スタイル**: セリフのない、整然としたエンジニアリング向けデザイン。
